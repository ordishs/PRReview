# PR Review — Phase 1, Plan 1: Project Scaffold

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up a building, testable macOS project — a `PRReviewCore` Swift package (6 targets) plus an XcodeGen-generated app target that launches to an empty sidebar+detail window.

**Architecture:** One local Swift package holds all headless logic as separate library targets (`PRReviewModels` + `ReviewStore`, `GitHubKit`, `WorktreeKit`, `DiffKit`, `ClaudeSessionKit`). A thin SwiftUI app target, defined declaratively in `project.yml` and generated by XcodeGen into `PRReview.xcodeproj` (gitignored), depends on the package. Package logic is tested with `swift test`; the app is built with `xcodebuild`.

**Tech Stack:** Swift 6 / Xcode 26.5, Swift Package Manager, Swift Testing (`import Testing`), SwiftUI, XcodeGen, Homebrew.

**Companion spec:** `docs/superpowers/specs/2026-05-27-pr-review-app-design.md`

**Plan sequence (Phase 1):** **1-scaffold (this)** → 2-reviewstore → 3-githubkit → 4-worktreekit → 5-diffkit → 6-app-integration.

## Task 2: Generate the macOS app target with XcodeGen

**Files:**
- Modify: `.gitignore` (append generated-project + derived-data entries)
- Create: `project.yml`
- Create: `App/PRReviewApp.swift`
- Create: `App/ContentView.swift`

- [ ] **Step 1: Ignore the generated project and build output**

Append these lines to `.gitignore`:

```gitignore
# XcodeGen-generated project (regenerate with `xcodegen generate`)
PRReview.xcodeproj/

# xcodebuild derived data used by this repo's build commands
DerivedData/
```

> `DerivedData/` may already be listed from the initial `.gitignore`; a duplicate line is harmless. The point is that the generated `.xcodeproj` is never committed — `project.yml` is the source of truth.

- [ ] **Step 2: Create the XcodeGen spec**

Create `project.yml`:

```yaml
name: PRReview
options:
  bundleIdPrefix: com.ordishs
  deploymentTarget:
    macOS: "14.0"
  createIntermediateGroups: true
packages:
  PRReviewCore:
    path: Core
targets:
  PRReview:
    type: application
    platform: macOS
    sources:
      - App
    dependencies:
      - package: PRReviewCore
        product: PRReviewModels
    settings:
      base:
        PRODUCT_BUNDLE_IDENTIFIER: com.ordishs.PRReview
        MARKETING_VERSION: "0.1.0"
        CURRENT_PROJECT_VERSION: "1"
        GENERATE_INFOPLIST_FILE: YES
        SWIFT_VERSION: "6.0"
        CODE_SIGNING_REQUIRED: NO
        CODE_SIGNING_ALLOWED: NO
schemes:
  PRReview:
    build:
      targets:
        PRReview: all
    run:
      config: Debug
```

> Code signing is disabled so `xcodebuild` and `swift test` succeed without a developer certificate. This is fine for a locally-run personal tool; we revisit signing if/when distribution matters.

- [ ] **Step 3: Create the app entry point**

Create `App/PRReviewApp.swift`:

```swift
import SwiftUI

@main
struct PRReviewApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
        }
    }
}
```

- [ ] **Step 4: Create the empty shell view**

Create `App/ContentView.swift`:

```swift
import SwiftUI

struct ContentView: View {
    var body: some View {
        NavigationSplitView {
            List {
                Text("No reviews yet")
                    .foregroundStyle(.secondary)
            }
            .navigationTitle("Reviews")
            .frame(minWidth: 240)
        } detail: {
            Text("Select a review")
                .foregroundStyle(.secondary)
                .font(.title3)
        }
        .frame(minWidth: 900, minHeight: 600)
    }
}
```

- [ ] **Step 5: Generate the Xcode project**

Run (from the repo root, where `project.yml` lives):

```bash
xcodegen generate
```

Expected: prints `⚙️  Generating project...` then `Created project at PRReview.xcodeproj`.

- [ ] **Step 6: Build the app**

Run:

```bash
xcodebuild -project PRReview.xcodeproj -scheme PRReview -configuration Debug -destination 'platform=macOS' -derivedDataPath DerivedData build CODE_SIGNING_ALLOWED=NO
```

Expected: ends with `** BUILD SUCCEEDED **`.

- [ ] **Step 7: Launch and eyeball it**

Run:

```bash
open DerivedData/Build/Products/Debug/PRReview.app
```

Expected: a window opens — a left sidebar titled "Reviews" reading "No reviews yet", and a detail area reading "Select a review". Quit the app (⌘Q) when satisfied.

- [ ] **Step 8: Commit**

```bash
git add .gitignore project.yml App
git commit -m "feat: add macOS app target via XcodeGen with empty shell"
```
